# Checklist Docs

## Discovery
- Confirmar fonte de verdade no código/config.
- Levantar público alvo e objetivo do documento.
- Identificar conteúdo obsoleto e lacunas.

## Implementação
- Escrever passos operacionais verificáveis.
- Incluir exemplos reais de comando/tool call.
- Padronizar termos e estrutura de seções.

## Validação
- Conferir paths, nomes e comandos descritos.
- Conferir consistência entre README e docs específicas.
- Conferir troubleshooting com ações concretas.

## Entrega
- Descrever mudanças e rationale editorial.
- Indicar documentos impactados.
- Registrar pendências documentais restantes.
